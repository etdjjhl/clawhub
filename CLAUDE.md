## Overview

This repository contains a single Bash script (`openclaw.sh`) — a multi-instance manager for OpenClaw, which runs as Docker containers. It provisions, starts, stops, updates, and inspects named OpenClaw instances, each with its own port, workspace, and config.

## Architecture

- **`openclaw.sh`** — the entire codebase. No build step, no dependencies beyond Bash, Docker, and `docker compose`.
- **`instances/<name>/`** — runtime directory per instance (gitignored), containing:
  - `.env` — `NAME`, `PORT`, `WORKSPACE_PATH`, `OPENCLAW_VERSION`
  - `docker-compose.yml` — generated by `write_compose()`
  - `config/` — persisted OpenClaw config (mounted into container at `/home/node/.openclaw`)
  - `workspace/` — user workspace (mounted into container)
- **Docker image**: `ghcr.io/openclaw/openclaw:<version>`, running as `uid 1000:1000`
- **Base port**: `18789`; each new instance auto-increments to the next free port
- **Docker Compose project name**: `openclaw-<name>`; gateway container name: `openclaw-<name>`

## Key Design Patterns

- `compose()` wraps `docker compose -p ... -f ... --env-file ...` for all Docker operations
- `write_compose()` / `write_env()` generate per-instance files at create time
- `patch_brand()` hot-patches the running container's UI bundle (via `docker cp` + `sed`) to show the instance name in the UI — this runs after every `create` and `update`
- The `openclaw-cli` service uses Docker `profiles: ["cli"]` so it only runs on-demand (for onboarding and config commands), never as a long-running container
- `load_env()` sources the instance `.env` to populate `NAME`, `PORT`, `WORKSPACE_PATH`, `OPENCLAW_VERSION` as shell variables

## Commands

```bash
./openclaw.sh create <name> [--workspace <path>] [--port <port>] [--version <tag>]
./openclaw.sh delete <name> [--purge]   # --purge removes files too
./openclaw.sh start  <name>
./openclaw.sh stop   <name>
./openclaw.sh status [<name>]           # all instances if no name given
./openclaw.sh list                      # tabular: name/port/status/version
./openclaw.sh info   <name>             # dashboard URL, token, stats
./openclaw.sh login  <name>             # bash shell in container
./openclaw.sh update <name> [--version <tag>]
./openclaw.sh version <name>
```
